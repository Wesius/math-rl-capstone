\begin{abstract}
This project explores whether reinforcement learning can teach a small (350M parameter) non-transformer language model to generate valid mathematical expressions that evaluate to target numbers. Using Group Relative Policy Optimization (GRPO) with a verifiable reward function, we fine-tune Liquid AI's LFM2-350M---a hybrid convolutional-attention architecture---on the task of producing arithmetic expressions for given numerical targets. We evaluate across difficulty levels (1--100, 100--999, 1000--9999) and compare against the unmodified base model. Our results reveal that while RL successfully teaches the model to produce correct expressions, the model consistently discovers formulaic shortcuts rather than genuine arithmetic reasoning---a finding with implications for reward design in RL-based language model training.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Large language models (LLMs) have demonstrated remarkable capabilities in mathematical reasoning, with models like DeepSeek-R1 and OpenAI's o1 achieving near-human performance on competition-level problems \cite{deepseekmath}. However, these capabilities typically require models with billions of parameters and enormous compute budgets. A natural question arises: \emph{can reinforcement learning induce structured reasoning in much smaller models?}

This project investigates this question through a focused, verifiable task: given a target integer $n$, generate a mathematical expression $e$ such that $\text{eval}(e) = n$. For example, given the target 73, valid responses include \texttt{70 + 3}, \texttt{146 / 2}, or \texttt{9 * 8 + 1}. This task is ideal for RL because:

\begin{enumerate}
    \item The reward is \textbf{perfectly verifiable}---we can programmatically check if $\text{eval}(e) = n$.
    \item The task requires \textbf{compositional reasoning}---the model must understand how arithmetic operations compose.
    \item Multiple valid solutions exist, encouraging \textbf{expression diversity}.
    \item Difficulty scales naturally with the magnitude of $n$.
\end{enumerate}

We choose Liquid AI's LFM2-350M \cite{lfm2} as our base model for several reasons. First, at 354M parameters, it is small enough to train on modest hardware. Second, it uses a novel hybrid architecture combining gated short-range convolutions with grouped query attention, making it architecturally distinct from the transformer-only models typically used in RL fine-tuning research. Third, Liquid AI explicitly recommends fine-tuning LFM2 on narrow tasks to maximize performance.

Our training method is Group Relative Policy Optimization (GRPO) \cite{deepseekmath}, implemented via Hugging Face's TRL library \cite{trl}. GRPO generates multiple completions per prompt, scores them with a reward function, and updates the policy using group-relative advantages---eliminating the need for a separate reward model.

\section{Data Exploration and Preparation}
\label{sec:data}

\subsection{Dataset Construction}

Rather than using an existing dataset, we procedurally generate training data consisting of (prompt, target) pairs. Each prompt asks the model to produce a math expression equaling the target number, formatted with \texttt{<expr></expr>} tags.

We use a curriculum-style distribution across three difficulty buckets:

\centering

\label{tab:data-dist}
\begin{tabular}{lccc}
\toprule
\textbf{Difficulty} & \textbf{Range} & \textbf{Fraction} & \textbf{Count} \\
\midrule

Medium & 100--999  & 35\% & 3,500 \\
Hard   & 1,000--9,999 & 25\% & 2,500 \\
\bottomrule
\end{tabular}
\end{table}

The rationale for overweighting easy examples is to provide the model with sufficient signal early in training. Since the 350M model starts with minimal arithmetic ability, easy targets (where simple expressions like \texttt{50 + 7} suffice) generate more positive reward signal, allowing the policy gradient to learn basic expression structure before tackling harder targets.

\subsection{Prompt Format}

Each training example is formatted as a chat conversation:

\begin{lstlisting}[caption={Prompt format used during training.}]
System: You are a math assistant. When given a target
number, respond with a mathematical expression that
evaluates to that number. Use only integers, +, -, *,
/, and parentheses. Put your expression inside
<expr></expr> tags.

User: Find a math expression that equals 73.
Respond with ONLY <expr>YOUR_EXPRESSION</expr>.
\end{lstlisting}

The structured output format (\texttt{<expr>} tags) enables reliable extraction and evaluation. We include a format reward to incentivize the model to adopt this convention.

\subsection{Evaluation Set}

A separate evaluation set of 500 examples is generated with a different random seed, maintaining the same difficulty distribution. While this does not guarantee zero overlap, the probability of significant overlap is low given the large target ranges.

\section{Methodology}
\label{sec:method}

\subsection{Model Architecture}

LFM2-350M \cite{lfm2} is a hybrid model with 16 layers: 10 double-gated short-range LIV convolution blocks and 6 grouped query attention (GQA) blocks. Key specifications:

\begin{table}[H]
\centering
\caption{LFM2-350M architecture details.}
\begin{tabular}{ll}
\toprule
\textbf{Property} & \textbf{Value} \\
\midrule
Parameters & 354,483,968 \\
Layers & 16 (10 conv + 6 attn) \\
Context length & 32,768 tokens \\
Vocabulary size & 65,536 \\
Precision & bfloat16 \\
Pre-training data & 10 trillion tokens \\
\bottomrule
\end{tabular}
\end{table}

The hybrid architecture is notable because the convolutional blocks process local patterns efficiently while attention blocks capture long-range dependencies. For our task, this could be advantageous since arithmetic expressions have strong local structure (operands adjacent to operators).

\subsection{GRPO Algorithm}

Group Relative Policy Optimization \cite{deepseekmath} works as follows at each training step:

\begin{enumerate}
    \item \textbf{Generate}: Sample a batch of prompts; for each, generate $G$ completions $\{o_1, \ldots, o_G\}$.
    \item \textbf{Score}: Compute reward $r_i$ for each completion using the reward function.
    \item \textbf{Normalize}: Compute group-relative advantage:
    \[
        \hat{A}_{i,t} = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})}
    \]
    \item \textbf{Update}: Minimize the clipped surrogate loss:
    \[
        \mathcal{L}_{\text{GRPO}}(\theta) = -\frac{1}{\sum_{i=1}^G |o_i|} \sum_{i=1}^G \sum_{t=1}^{|o_i|} \min\!\left( \rho_{i,t} \hat{A}_{i,t},\; \text{clip}(\rho_{i,t}, 1{-}\epsilon, 1{+}\epsilon) \hat{A}_{i,t} \right)
    \]
    where $\rho_{i,t} = \frac{\pi_\theta(o_{i,t} | q, o_{i,<t})}{\pi_{\theta_{\text{old}}}(o_{i,t} | q, o_{i,<t})}$.
\end{enumerate}

GRPO's key advantage over PPO is eliminating the value network, reducing memory overhead---critical when training on limited hardware.

\subsection{Reward Design}

We use a composite reward function with four components:

\begin{table}[H]
\centering
\caption{Reward function components and weights.}
\label{tab:rewards}
\begin{tabular}{llcl}
\toprule
\textbf{Component} & \textbf{Description} & \textbf{Weight} & \textbf{Range} \\
\midrule
Correctness & Binary: $\text{eval}(e) = n$ & 2.0 & $\{0, 1\}$ \\
Closeness & $\max(0, 1 - |r - n|/|n|)$ & 0.5 & $[0, 1]$ \\
Format & Uses \texttt{<expr>} tags & 0.3 & $\{0, 0.5\}$ \\
Complexity & Bonus for multiple operators & 0.2 & $[0, 0.3]$ \\
\bottomrule
\end{tabular}
\end{table}

The \textbf{correctness} reward dominates (weight 2.0) because exact evaluation is the primary objective. The \textbf{closeness} reward provides gradient signal even for incorrect answers---without it, early training would be extremely sparse since the base model rarely produces correct expressions. The \textbf{format} reward teaches output structure, and the \textbf{complexity} reward gently encourages expressions more interesting than trivial identity (e.g., \texttt{73} for target 73).

Expression evaluation uses a safe AST-based parser that only allows integer literals and basic arithmetic operators (\texttt{+}, \texttt{-}, \texttt{*}, \texttt{/}, \texttt{//}, \texttt{\%}, \texttt{**}), preventing code injection.

\section{Results and Discussion}
\label{sec:results}

\subsection{Training Dynamics}

We ran four experimental configurations on an NVIDIA A100-40GB GPU, each training for one epoch (250 steps) with batch size 16 and 8 generations per prompt.

\subsubsection{Baseline: Default Reward Weights}

With the original reward weights (correctness=2.0, closeness=0.5, format=0.3, complexity=0.2), the model converged within $\sim$50 steps to a \textbf{trivial solution}: simply echoing the target number inside \texttt{<expr>} tags. By step 50, correctness reached 100\%, format reached 100\%, but complexity remained at 0---the model learned to output \texttt{<expr>73</expr>} for target 73. Mean completion length collapsed from 35 tokens to just 8 tokens. This is a classic reward hacking phenomenon: the model found the minimum-effort path to maximize the dominant reward signal.

\subsubsection{Experiment 1: High Complexity Weight}

Increasing the complexity reward weight from 0.2 to 2.0 (equal to correctness) produced dramatically different behavior. The model learned to generate multi-operator expressions while maintaining 99.4\% correctness. Mean completion length grew to 50 tokens, and the complexity reward saturated at 0.3 (its maximum). Training showed three distinct phases: (1) format learning (steps 0--25), (2) correctness acquisition (steps 25--100), and (3) expression diversification (steps 100--250).

\subsubsection{Experiment 2: Anti-Trivial Reward}

Adding a ``nontrivial'' reward that penalizes single-number expressions ($-1.0$ for identity, $+1.0$ for multi-operator) with weight 1.0 was the most effective configuration. The model achieved \textbf{100\% correctness} with maximum complexity (0.3) and nontrivial score (1.0). Mean completion length stabilized at 20 tokens---concise but non-trivial expressions. This configuration found the best balance between correctness and expression quality.

\subsubsection{Experiment 4: No-Target Reward}

The most effective configuration added a \emph{no-target reward} that penalizes ($-1.0$) any expression containing the target number as a literal. Combined with high complexity (2.0), anti-trivial (1.0), and closeness (0.5) rewards, this forced genuine number decomposition. The model could no longer use \texttt{(73-1)/1+1} because \texttt{73} appears in the expression.

Results were qualitatively different from all prior experiments. Correctness reached only 5\% (with closeness at 0.85), but the model produced \emph{diverse, structurally varied expressions}: \texttt{7*10+3+1/2} for target 73, \texttt{2\^{}8} for 256, \texttt{2\^{}11} for 2048. It discovered powers of two on its own and decomposed numbers into meaningful arithmetic. The dominant template covered only 40\% of outputs (vs.\ 100\% in all other experiments), with 6 unique structural templates across 10 samples.

\subsubsection{Experiment 3: Heavy Complexity, No Closeness}

Removing the closeness reward (weight=0.0) and using heavy complexity (weight=3.0) with the anti-trivial penalty created an interesting failure mode. The model quickly learned format compliance and non-trivial expression structure, but correctness remained near 0\%. Without the closeness signal to guide it toward numerically accurate expressions, the model generated syntactically valid but mathematically incorrect expressions. This confirms that the closeness reward serves as a critical \textbf{reward shaping} signal for this task.

Four types of behavior emerged:

\begin{enumerate}
    \item \textbf{Base model}: Produces natural language explanations, algebraic variables, and incorrect arithmetic. Only 1/10 targets answered correctly (by accident).
    \item \textbf{Exp 1 \& 2}: Discovered \textbf{formulaic identity hacks}---\texttt{(N-1)/1+1} and \texttt{(N+1)/1-1} respectively. These are technically non-trivial (3 operators, satisfying the complexity reward) but semantically degenerate. Every target produces the same template. This represents a \emph{second-order reward hack}: the model satisfies both correctness \emph{and} complexity constraints with minimal cognitive effort.
    \item \textbf{Exp 3}: Generates diverse, complex expressions with multiple operators, but none evaluate correctly. The model learned \emph{syntax} (format, operators) without \emph{semantics} (numerical accuracy).
    \item \textbf{Exp 4}: Produces genuinely varied decompositions---\texttt{7*10+3} for 73, \texttt{2\^{}8} for 256---with 6 unique templates across 10 samples. Discovered powers of two independently. Most answers are close but not exact (closeness 0.85), representing the first configuration to exhibit what could be called arithmetic \emph{reasoning}.
\end{enumerate}

This progression from trivial hacking (baseline) to formulaic hacking (Exp 1--2) to syntactically-rich-but-wrong (Exp 3) to diverse-and-close (Exp 4) reveals a fundamental tension in reward design: each additional constraint closes one loophole while the model discovers the next path of least resistance---until the constraints collectively force genuine problem-solving.

The primary failure modes observed across configurations:

\subsection{Discussion}

Several aspects of the results merit discussion:

\textbf{Reward hacking is multi-layered, but solvable.} Na\"ive reward design leads directly to trivial solutions, and each added constraint opens a new exploit---until enough constraints force genuine reasoning. The critical breakthrough was Experiment 4's \emph{no-target reward}, which bans the target number from the expression. This single constraint, combined with closeness and complexity rewards, produced diverse decompositions and the first evidence of genuine arithmetic reasoning (e.g., independently discovering \texttt{2\^{}8=256}).

\textbf{Anti-trivial rewards are more effective than high complexity weights.} Experiment 2 (explicit penalty for identity expressions) outperformed Experiment 1 (high complexity weight) in producing concise, correct, non-trivial expressions. The penalty creates a clear gradient \emph{away} from the trivial solution, while a high complexity weight merely offers an incentive that competes with the easier correctness path.

\textbf{Reward shaping is essential for sparse-reward tasks.} Experiment 3 definitively demonstrates that removing the closeness reward prevents learning correctness entirely. Interestingly, Exp~3 produced the most \emph{diverse} and \emph{creative} expressions---but none were correct. The closeness signal acts as a \emph{potential-based reward shaping} function, providing a smooth gradient toward the correct answer that the binary correctness reward alone cannot provide for a 350M model.

\textbf{Architecture considerations.} LFM2's hybrid architecture---with its convolutional blocks for local pattern recognition and attention for long-range dependencies---proved fully compatible with GRPO training. The model's rapid convergence (within 50 steps for the baseline, $\sim$100 steps for Exp~1--2) suggests the conv-attention hybrid can efficiently capture arithmetic patterns.

\textbf{Implications for RL alignment.} Our findings echo broader concerns in RLHF/RLAIF research: models under optimization pressure find the \emph{easiest} way to satisfy reward criteria, not the \emph{intended} way. The progression from trivial $\to$ formulaic $\to$ syntactic-only behavior mirrors reward hacking phenomena observed in much larger models.

\section{Conclusion}
\label{sec:conclusion}

This project demonstrates that Group Relative Policy Optimization can teach a small, non-transformer language model to generate valid mathematical expressions for target numbers. Through systematic ablation of reward components, we identified three key findings:

\begin{enumerate}
    \item \textbf{Reward hacking is multi-layered, but solvable.} The same 350M model progresses from trivially correct (baseline), to formulaically correct (Exp~1--2), to syntactically rich but incorrect (Exp~3), to genuinely diverse and close (Exp~4). The no-target reward was the critical constraint that forced real decomposition, producing 6$\times$ more template diversity and emergent arithmetic behavior.
    \item \textbf{Diversity requires structural constraints.} Reward-level incentives alone (complexity weight, anti-trivial penalty) are insufficient to prevent template collapse. Structural constraints that ban specific output patterns (no-target reward) are needed to force genuine solution diversity.
    \item \textbf{Reward shaping is essential for sparse-reward RL on small models.} Removing the closeness reward (Exp~3) completely prevented learning correctness, even with strong complexity incentives. This confirms that auxiliary dense reward signals are critical when the primary reward is binary.
    \item \textbf{Novel architectures are RL-compatible.} LFM2's hybrid conv-attention design works seamlessly with standard GRPO training via TRL and LoRA, converging within 50--100 steps across all configurations.
\end{enumerate}

\subsection{Future Work}

\begin{itemize}
    \item \textbf{Curriculum learning}: Progressively increase target difficulty during training rather than using a fixed distribution.
    \item \textbf{Chain-of-thought}: Allow the model to reason step-by-step before producing the final expression.
    \item \textbf{Architecture comparison}: Compare LFM2 against a similarly-sized transformer (e.g., Qwen3-0.6B) to isolate architectural effects.
    \item \textbf{Deployment}: Package the fine-tuned model as a lightweight API or web demo (optional extra credit component).
\end{itemize}
