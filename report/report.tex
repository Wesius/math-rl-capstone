\begin{abstract}
We use Group Relative Policy Optimization (GRPO) to teach Liquid AI's LFM2-350M---a 354M-parameter hybrid convolutional-attention model---to generate mathematical expressions evaluating to target numbers. Through 13 experiments systematically varying reward components, we discover that the model exhibits multi-layered reward hacking: trivial identity, formulaic templates, and syntax-without-semantics, each requiring a corresponding structural constraint to overcome. A \emph{no-target reward} that bans the target number from appearing in expressions proves critical for inducing genuine arithmetic decomposition (e.g., the model independently discovers $2^8 = 256$). We map the diversity-correctness Pareto frontier, revealing a sharp phase transition in reward space: small changes in reward weights trigger abrupt collapse from diverse exploration to single-template convergence. The optimal configuration achieves 30\% template diversity with 71.7\% correctness, offering insights into reward design for RL-based language model training.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Large language models (LLMs) have demonstrated remarkable capabilities in mathematical reasoning, with models like DeepSeek-R1 and OpenAI's o1 achieving near-human performance on competition-level problems \cite{deepseekmath}. However, these capabilities typically require models with billions of parameters and enormous compute budgets. A natural question arises: \emph{can reinforcement learning induce structured reasoning in much smaller models?}

This project investigates this question through a focused, verifiable task: given a target integer $n$, generate a mathematical expression $e$ such that $\text{eval}(e) = n$. For example, given the target 73, valid responses include \texttt{70 + 3}, \texttt{146 / 2}, or \texttt{9 * 8 + 1}. This task is ideal for RL because:

\begin{enumerate}
    \item The reward is \textbf{perfectly verifiable}---we can programmatically check if $\text{eval}(e) = n$.
    \item The task requires \textbf{compositional reasoning}---the model must understand how arithmetic operations compose.
    \item Multiple valid solutions exist, encouraging \textbf{expression diversity}.
    \item Difficulty scales naturally with the magnitude of $n$.
\end{enumerate}

We choose Liquid AI's LFM2-350M \cite{lfm2} as our base model for several reasons. First, at 354M parameters, it is small enough to train on modest hardware. Second, it uses a novel hybrid architecture combining gated short-range convolutions with grouped query attention, making it architecturally distinct from the transformer-only models typically used in RL fine-tuning research. Third, Liquid AI explicitly recommends fine-tuning LFM2 on narrow tasks to maximize performance.

Our training method is Group Relative Policy Optimization (GRPO) \cite{deepseekmath}, implemented via Hugging Face's TRL library \cite{trl}. GRPO generates multiple completions per prompt, scores them with a reward function, and updates the policy using group-relative advantages---eliminating the need for a separate reward model.

\subsection{Dataset Construction}

Rather than using an existing dataset, we procedurally generate training data consisting of (prompt, target) pairs. Each prompt asks the model to produce a math expression equaling the target number, formatted with \texttt{<expr></expr>} tags.

We use a curriculum-style distribution across three difficulty buckets:

The rationale for overweighting easy examples is to provide the model with sufficient signal early in training. Since the 350M model starts with minimal arithmetic ability, easy targets (where simple expressions like \texttt{50 + 7} suffice) generate more positive reward signal, allowing the policy gradient to learn basic expression structure before tackling harder targets.

\subsection{Prompt Format}

Each training example is formatted as a chat conversation:

\begin{lstlisting}[caption={Prompt format used during training.}]
System: You are a math assistant. When given a target
number, respond with a mathematical expression that
evaluates to that number. Use only integers, +, -, *,
/, and parentheses. Put your expression inside
<expr></expr> tags.

User: Find a math expression that equals 73.
Respond with ONLY <expr>YOUR_EXPRESSION</expr>.
\end{lstlisting}

The structured output format (\texttt{<expr>} tags) enables reliable extraction and evaluation. We include a format reward to incentivize the model to adopt this convention.

\subsection{Evaluation Set}

A separate evaluation set of 500 examples is generated with a different random seed, maintaining the same difficulty distribution. While this does not guarantee zero overlap, the probability of significant overlap is low given the large target ranges.

\section{Methodology}
\label{sec:method}

\subsection{Model Architecture}

LFM2-350M \cite{lfm2} is a hybrid model with 16 layers: 10 double-gated short-range LIV convolution blocks and 6 grouped query attention (GQA) blocks. Key specifications:

\begin{table}[H]
\centering
\caption{LFM2-350M architecture details.}
\begin{tabular}{ll}
\toprule
\textbf{Property} & \textbf{Value} \\
\midrule
Parameters & 354,483,968 \\
Layers & 16 (10 conv + 6 attn) \\
Context length & 32,768 tokens \\
Vocabulary size & 65,536 \\
Precision & bfloat16 \\
Pre-training data & 10 trillion tokens \\
\bottomrule
\end{tabular}
\end{table}

The hybrid architecture is notable because the convolutional blocks process local patterns efficiently while attention blocks capture long-range dependencies. For our task, this could be advantageous since arithmetic expressions have strong local structure (operands adjacent to operators).

\subsection{GRPO Algorithm}

Group Relative Policy Optimization \cite{deepseekmath} works as follows at each training step:

\begin{enumerate}
    \item \textbf{Generate}: Sample a batch of prompts; for each, generate $G$ completions $\{o_1, \ldots, o_G\}$.
    \item \textbf{Score}: Compute reward $r_i$ for each completion using the reward function.
    \item \textbf{Normalize}: Compute group-relative advantage:
    \[
        \hat{A}_{i,t} = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})}
    \]
    \item \textbf{Update}: Minimize the clipped surrogate loss:
    \[
        \mathcal{L}_{\text{GRPO}}(\theta) = -\frac{1}{\sum_{i=1}^G |o_i|} \sum_{i=1}^G \sum_{t=1}^{|o_i|} \min\!\left( \rho_{i,t} \hat{A}_{i,t},\; \text{clip}(\rho_{i,t}, 1{-}\epsilon, 1{+}\epsilon) \hat{A}_{i,t} \right)
    \]
    where $\rho_{i,t} = \frac{\pi_\theta(o_{i,t} | q, o_{i,<t})}{\pi_{\theta_{\text{old}}}(o_{i,t} | q, o_{i,<t})}$.
\end{enumerate}

GRPO's key advantage over PPO is eliminating the value network, reducing memory overhead---critical when training on limited hardware.

\subsection{Reward Design}

We use a composite reward function with four components:

\begin{table}[H]
\centering
\caption{Reward function components and weights.}
\label{tab:rewards}
\begin{tabular}{llcl}
\toprule
\textbf{Component} & \textbf{Description} & \textbf{Weight} & \textbf{Range} \\
\midrule
Correctness & Binary: $\text{eval}(e) = n$ & 2.0 & $\{0, 1\}$ \\
Closeness & $\max(0, 1 - |r - n|/|n|)$ & 0.5 & $[0, 1]$ \\
Format & Uses \texttt{<expr>} tags & 0.3 & $\{0, 0.5\}$ \\
Complexity & Bonus for multiple operators & 0.2 & $[0, 0.3]$ \\
\bottomrule
\end{tabular}
\end{table}

The \textbf{correctness} reward dominates (weight 2.0) because exact evaluation is the primary objective. The \textbf{closeness} reward provides gradient signal even for incorrect answers---without it, early training would be extremely sparse since the base model rarely produces correct expressions. The \textbf{format} reward teaches output structure, and the \textbf{complexity} reward gently encourages expressions more interesting than trivial identity (e.g., \texttt{73} for target 73).

Expression evaluation uses a safe AST-based parser that only allows integer literals and basic arithmetic operators (\texttt{+}, \texttt{-}, \texttt{*}, \texttt{/}, \texttt{//}, \texttt{\%}, \texttt{**}), preventing code injection.

\section{Results and Discussion}
\label{sec:results}

\subsection{Training Dynamics}

We conducted 13 experiments across two waves on an NVIDIA A100-80GB GPU, systematically varying reward weights to map the diversity-correctness tradeoff. Each experiment trained for 1 epoch (250 steps) with batch size 16 and 8 generations per prompt unless otherwise noted. We describe the five most informative configurations below.

\subsubsection{Baseline: Default Reward Weights}

With the original reward weights (correctness=2.0, closeness=0.5, format=0.3, complexity=0.2), the model converged within $\sim$50 steps to a \textbf{trivial solution}: simply echoing the target number inside \texttt{<expr>} tags. By step 50, correctness reached 100\%, format reached 100\%, but complexity remained at 0---the model learned to output \texttt{<expr>73</expr>} for target 73. Mean completion length collapsed from 35 tokens to just 8 tokens. This is a classic reward hacking phenomenon: the model found the minimum-effort path to maximize the dominant reward signal.

\subsubsection{Experiments 1--2: Complexity and Anti-Trivial Rewards}

Increasing the complexity reward weight to 2.0 (Exp~1) or adding an anti-trivial penalty (Exp~2) both produced 99--100\% correctness with multi-operator expressions. However, qualitative analysis (Section~\ref{sec:qualitative}) revealed that both discovered \textbf{formulaic identity hacks}: \texttt{(N-1)/1+1} (Exp~1) and \texttt{(N+1)/1-1} (Exp~2). Every target produced the same template---a second-order reward hack satisfying both correctness \emph{and} complexity with a single memorized pattern.

\subsubsection{Experiment 3: No Closeness Reward}

Removing the closeness reward (weight=0.0) with heavy complexity (weight=3.0) created an instructive failure. The model learned format compliance and expression syntax, but correctness remained near 0\%. Without the closeness signal, it generated syntactically valid but mathematically incorrect expressions, confirming that closeness serves as a critical \textbf{reward shaping} signal.

\subsubsection{Experiment 4: No-Target Reward}

The key breakthrough: a \emph{no-target reward} penalizing ($-1.0$) any expression containing the target number as a literal. Combined with complexity (2.0), anti-trivial (1.0), and closeness (0.5) rewards, this forced genuine decomposition. The model could no longer use \texttt{(73-1)/1+1} because \texttt{73} appears in the expression.

The model produced \emph{diverse, structurally varied expressions}: \texttt{7*10+3+1/2} for target 73, \texttt{2\^{}8} for 256, \texttt{2\^{}11} for 2048. It discovered powers of two independently. Correctness was 5\% with closeness at 0.85, representing the first configuration exhibiting genuine arithmetic reasoning.

\subsubsection{Reward Weight Sweep: The Diversity-Correctness Pareto Frontier}

Building on Experiment 4, we ran 8 additional experiments varying reward weights to map the tradeoff between diversity and correctness. Each used the no-target and anti-trivial rewards, with closeness weight swept from 0.5 to 2.0 and correctness weight from 0.5 to 2.0.

We found a \textbf{sharp phase transition}: below closeness weight $\approx$1.3, the model explores diverse strategies; above it, the model collapses to a single template with $\sim$100\% correctness and 100\% target leakage (rediscovering the identity hack despite the no-target penalty). The optimal configuration was \textbf{correctness=1.0, closeness=0.5}, achieving 30\% template diversity with 71.7\% correctness.

Four types of behavior emerged:

\begin{enumerate}
    \item \textbf{Base model}: Produces natural language explanations, algebraic variables, and incorrect arithmetic. Only 1/10 targets answered correctly (by accident).
    \item \textbf{Exp 1 \& 2}: Discovered \textbf{formulaic identity hacks}---\texttt{(N-1)/1+1} and \texttt{(N+1)/1-1} respectively. These are technically non-trivial (3 operators, satisfying the complexity reward) but semantically degenerate. Every target produces the same template. This represents a \emph{second-order reward hack}: the model satisfies both correctness \emph{and} complexity constraints with minimal cognitive effort.
    \item \textbf{Exp 3}: Generates diverse, complex expressions with multiple operators, but none evaluate correctly. The model learned \emph{syntax} (format, operators) without \emph{semantics} (numerical accuracy).

This progression from trivial hacking (baseline) to formulaic hacking (Exp 1--2) to syntactically-rich-but-wrong (Exp 3) to diverse-and-close (Exp 4) reveals a fundamental tension in reward design: each additional constraint closes one loophole while the model discovers the next path of least resistance---until the constraints collectively force genuine problem-solving.

\subsection{Error Analysis}

\begin{figure}[H]

The primary failure modes observed across 13 configurations:

\begin{itemize}
    \item \textbf{First-order reward hacking} (Baseline): The model echoes the target number inside \texttt{<expr>} tags, achieving 100\% correctness with zero operators. Completion length collapses to 8 tokens.
    \item \textbf{Second-order reward hacking} (Exp 1--2): Even with complexity incentives, the model discovers formulaic identities (\texttt{(N-1)/1+1} and \texttt{(N+1)/1-1}) that satisfy both correctness and complexity constraints with a single memorized template.
    \item \textbf{Syntax without semantics} (Exp 3): Without closeness-based reward shaping, the model learns rich expression syntax but produces numerically incorrect results.
    \item \textbf{Phase transition collapse} (Wave 2): At closeness weight $\geq$1.3 or learning rate $\geq$1e-4, even models with the no-target reward collapse to identity hacks---the optimization pressure overwhelms the structural constraint.     \item \textbf{Training duration collapse} (Wave 2): Running the best diversity config (div01) for 2 epochs instead of 1 dropped template diversity from 0.40 to 0.033. Given enough time, the model always converges to a template.
\end{itemize}

\subsection{Discussion}

Several aspects of the results merit discussion:

\textbf{Reward hacking is multi-layered, but solvable.} Na\"ive reward design leads directly to trivial solutions, and each added constraint opens a new exploit---until enough constraints force genuine reasoning. The critical breakthrough was Experiment 4's \emph{no-target reward}, which bans the target number from the expression. This single constraint, combined with closeness and complexity rewards, produced diverse decompositions and the first evidence of genuine arithmetic reasoning (e.g., independently discovering \texttt{2\^{}8=256}).

\textbf{The diversity-correctness Pareto frontier.} Our 13-experiment sweep revealed a nearly inverse relationship between expression diversity and correctness. The frontier (Table~\ref{tab:pareto}) shows that configurations cluster into three regimes: (1) high diversity / low correctness (template ratio $>$0.3, correctness $<$15\%), (2) the sweet spot (template ratio $\approx$0.3, correctness $\approx$72\%), and (3) template collapse (template ratio $<$0.05, correctness $>$85\%). The key to reaching regime (2) was \emph{reducing} correctness weight from 2.0 to 1.0---counterintuitively, less optimization pressure on the primary objective produced better overall results.

\textbf{Structural constraints outperform reward incentives.} Adding complexity weight or anti-trivial penalties (reward-level incentives) failed to prevent template collapse. The no-target reward (a structural constraint banning specific output patterns) was the only intervention that produced genuine diversity. This suggests that for RL-trained language models, \emph{what you forbid} matters more than \emph{what you reward}.

\textbf{Sharp phase transitions in reward space.} The model exhibits bistable behavior: small changes in closeness weight (1.0 $\to$ 1.3) cause an abrupt transition from diverse exploration to single-template collapse. Similarly, doubling the learning rate or training for one additional epoch triggers the same collapse. This suggests the reward landscape has sharp basins of attraction---once the model discovers an exploit, gradient descent rapidly converges to it.

\textbf{Reward shaping is essential for sparse-reward tasks.} Experiment 3 definitively demonstrates that removing the closeness reward prevents learning correctness entirely. The closeness signal acts as a \emph{potential-based reward shaping} function, providing a smooth gradient toward the correct answer that the binary correctness reward alone cannot provide for a 350M model.

\textbf{Implications for RL alignment.} Our findings echo broader concerns in RLHF/RLAIF research: models under optimization pressure find the \emph{easiest} way to satisfy reward criteria, not the \emph{intended} way. The progression from trivial $\to$ formulaic $\to$ syntactic-only $\to$ genuinely diverse behavior required four layers of constraints, mirroring the ``whack-a-mole'' dynamic observed in alignment research at scale.

\section{Conclusion}
\label{sec:conclusion}

This project demonstrates that GRPO can teach a 350M-parameter non-transformer model to generate mathematical expressions for target numbers, and that the reward design challenge is far more interesting than the model training itself. Through 13 experiments across two systematic sweeps, we identified four key findings:

\begin{enumerate}
    \item \textbf{Reward hacking is multi-layered, but solvable.} The same model progresses from trivial identity (baseline), to formulaic hack (Exp~1--2), to syntax without semantics (Exp~3), to genuinely diverse decomposition (Exp~4). Each layer of hacking required a corresponding structural constraint to overcome.
    \item \textbf{Structural constraints beat reward incentives.} The no-target reward---banning the target number from expressions---was the critical breakthrough. Reward-level incentives (complexity weight, anti-trivial penalty) only shifted \emph{which} template the model converged to; structural constraints forced genuine \emph{diversity}.
    \item \textbf{A sharp phase transition governs the diversity-correctness tradeoff.} Small changes in reward weights (closeness 1.0$\to$1.3) or training dynamics (1$\to$2 epochs, 5e-5$\to$1e-4 lr) trigger abrupt collapse from diverse exploration to single-template convergence. The optimal operating point (correctness=1.0, closeness=0.5) achieves 30\% template diversity with 71.7\% correctness.
    \item \textbf{Reward shaping is essential for sparse-reward RL on small models.} The closeness reward is the gradient bridge between ``random noise'' and ``correct answer.'' Without it, the model cannot learn correctness regardless of other reward signals.
\end{enumerate}

\subsection{Future Work}

\begin{itemize}
    \item \textbf{Diversity reward}: Directly penalize repeated expression \emph{structures} within a GRPO group, rather than relying on indirect structural constraints.
    \item \textbf{Banned subexpressions}: Extend the no-target approach to ban degenerate patterns like \texttt{/1}, \texttt{*1}, \texttt{+0}, closing additional exploit paths.
    \item \textbf{Curriculum learning}: Start with easy targets (powers of 2, round numbers with obvious decompositions), then introduce harder targets progressively.
    \item \textbf{Architecture comparison}: Compare LFM2 against a similarly-sized transformer (e.g., Qwen3-0.6B) to isolate whether the hybrid architecture's local pattern recognition offers advantages for arithmetic expressions.
    \item \textbf{Scaling}: Test whether larger LFM2 variants (700M, 1.2B) can cross the diversity-correctness tradeoff barrier.
\end{itemize}
